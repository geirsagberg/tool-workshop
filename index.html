Hva er et utviklerverktøy?

  • Løs små eller store hverdagsproblemer
  • Automatiserer, effektiviserer, eller visualiserer arbeidsoppgaver
  • Kan være alt fra et lite script til en full webapp

Hvordan gå fram?

  1. Kartlegg oppgaver du gjør ofte eller som tar tid
  2. Bryt ned i konkrete steg
  3. Finn variasjoner/parametre
  4. Avklar autentisering/tilgang
  5. Skissér enkel løsning
  6. Bygg og iterer
  7. Del og forbedre

Eksempler på scripts (CLI)

  • Bash, Python, Node.js, PowerShell, osv.
  • Automatisering, filbehandling, API-kall
  • Ofte raskest å komme i gang
  • CLI-biblioteker: Click (Python), Commander (Node), Clap (Rust), fzf (bash)
  • Bruk .env-filer for konfig
  • Prompts og interaktivitet (fzf, read, questionary, inquirer)

Tips for CLI-verktøy

  • Velg språk etter behov: Bash for enkle oppgaver, Python/Node for mer logikk
  • Bruk CLI-biblioteker for bedre UX
  • fzf gir raske valg i terminalen
  • Ikke hardkod sensitive data – bruk .env
  • Gi alltid en help-opsjon og gode feilmeldinger

Eksempler på SPA-verktøy

  • Dashboards, visualisering, intern selvbetjening
  • Lages ofte med React, Vue, Svelte m.m.
  • Bruk komponentbibliotek for rask UI-utvikling
  • localStorage for enkel lagring
  • Unngå routing hvis mulig
  • Enkel autentisering: bruk ditt eget token

Tips for GUI/SPA

  • Velg ett komponentbibliotek og lær det godt
  • Litt styling gir stor gevinst
  • localStorage for lagring av småting
  • Unngå routing om det ikke trengs
  • React + TypeScript + Tanstack Query + fetch dekker mye
  • Dropp auth-flyt hvis du kan – bruk tokenfelt

Eksempler på full-stack/hybrid

  • Next.js, Remix, TanStack Start – både frontend og backend
  • Verktøy som kaller shell-kommandoer via API
  • Desktop-apper: Electron, Tauri
  • API først – frontend og CLI kan bruke samme API

Tips for full-stack/hybrid

  • Start med en enkel template for ditt favoritt-rammeverk
  • Bruk execa for å kjøre kommandoer (Node)
  • Valider og saniter input før du kjører shell
  • Lag API-endepunkter for gjenbruk
  • Logg handlinger/serverresultater
  • Start uten database – legg til senere hvis nødvendig
  • Hold autentisering enkelt for interne verktøy

Deling og publisering

  • Del på Github, interne repo, npm, pip osv.
  • Lag kort dokumentasjon eller eksempler
  • Vurder om det er nyttig for flere
  • Enkelt å utvide/forbedre over tid

Workshop: Bygg et eget verktøy!

  • Velg noe du ofte gjør – automatiser det
  • CLI, SPA eller full-stack, valgfritt nivå
  • Jobb alene eller i par
  • Demo og del erfaringer til slutt
